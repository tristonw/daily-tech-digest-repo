主持人A：哈喽大家好，欢迎收听每日科技播客，我是晓宇。今天是二零二六年六月九号，星期二。昨天我们聊了开发者群体的"AI 情绪三件套"集中爆发、Meta 官方确认数千 Instagram 账号被 AI 客服 Agent 攻击、Agent 中台 SKU 细分进入下一阶段。今天的主线非常清楚——三大 AI 平台同日交卷。Apple 把 Siri 重塑为"Siri AI"，底层调用 Google 模型；Google 把 Gemini 3.5 加 Antigravity 推进 NotebookLM；OpenAI 准备给 ChatGPT 做"Chat is dead"级别的大改版，走 Super App 路线。三大 AI 平台同日动手，意味着 2026 下半年的 AI 产品形态会从"聊天框"全面跨进"系统级智能体 + 多模型路由 + 高毛利垂类应用"——这是个分水岭日子。

嘉宾B：大家好，我是思琪。今天的五条主线我先把骨架立起来——第一条是三大 AI 平台同日交卷，"聊天框时代"今天正式结束。第二条是小米 MiMo-v2.5-Pro-UltraSpeed 把 1T 参数模型的推理速度做到 1000 tokens/秒，HN 热度四百五十五、评论三百零九，中国厂商在"超大模型 + 超低延迟"双维度上首次和海外厂商同台。第三条是 Microsoft 73 个包再次被植入凭证窃取器，"被 AI Agent 一打开就自我复制"——这是昨天 Meta 事件之后的第二刀，Agent 供应链安全今天升级为"包级别污染"。第四条是 xAI 被指越来越像数据中心 REIT 而不是前沿实验室，热度三百四十二、评论二百六十三，配合"AI Is Slowing Down"长文三百二十二热度、三百五十五评论——AI 估值叙事第一次出现可观测的二分。第五条是 Agent SKU 与开源 NotebookLM 替代——goose、taste-skill、open-notebook、CopilotKit 这套生态今天集体进 Trending，是企业 AI 平台落地的开源主力。

主持人A：好，我们一条一条来。先讲今天最重的——三大 AI 平台同日交卷。

主持人A：思琪，先给大家完整画一下这三家今天分别做了什么。

嘉宾B：好。三家今天的动作虽然不是真的"同步"发布、但媒体周期叠在一起，效果就是"同日交卷"。第一家是 Apple——"Siri AI" 正式发布，秋季上线。三个看点：一是更口语化的语音助理；二是双层模型架构——前端是 Apple 自家的小模型负责路由、个性化、隐私推理，后端调用 Google 模型负责前沿能力；三是 Apple 第一次公开承认"模型层不是我们做的"。这是 Apple 史上对"模型即代工"的最明确表态。

主持人A：Google 这边是 NotebookLM 接 Gemini 3.5。

嘉宾B：对。Google 把 Gemini 3.5 加 Antigravity 推进 NotebookLM，但只对 AI Ultra 和企业账户开放。Google 的逻辑是把"研究 / 笔记"作为 Gemini 高端订阅的差异化场景——你想用最新最强的 Gemini 做研究助手，就必须订阅高价的 Ultra 套餐。这是 Google 把知识管理 Agent 推成"高价订阅产品"的关键一步。对企业的含义是——你想给员工买"研究助手"，就要按用户买 Google Ultra；如果不想买，开源替代方案是 open-notebook。

主持人A：OpenAI 这边是 ChatGPT 大改版加 Super App 化。

嘉宾B：是的。Ars Technica 直接打了一个非常重的标题——"Chat is dead"。OpenAI 准备在潜在 IPO 之前把 ChatGPT 从"聊天机器人"重塑为"高毛利产品的入口"。配合 MIT Technology Review 那条"OpenAI Super App"补充，OpenAI 想把 ChatGPT 做成微信式的超级 App——承担"购物 + 客服 + 工作 + 娱乐"多场景入口。这是把 ChatGPT 从"问答工具"升级为"Agent 操作系统 + 应用商店"的关键转折。

主持人A：三家放在一起看，有什么共同的认知？

嘉宾B：共同的认知是——"聊天框"作为唯一交互形式已经到顶。下一阶段的差异化来自三件事：在哪个入口出现、给哪类任务服务、用什么形态呈现。Apple 走的是"系统级语音入口 + 多模型路由"路线；Google 走的是"知识管理 Agent + 高价订阅"路线；OpenAI 走的是"AI 操作系统 + 应用商店"路线。三条路线背后的反思一致——你不能再用一个聊天框打天下了。

主持人A：这对企业内部 AI 应用规划意味着什么？

嘉宾B：意味着一件事——内部 AI 应用的产品设计应该立即跳出"聊天框默认形态"。按业务任务设计专用的 Agent 入口与界面——可以是语音、可以是表单、可以是生成式 UI、可以是 Slack 嵌入、可以是 IDE 嵌入。今天 GitHub Trending 上 CopilotKit 三万四千多星、open-notebook 两万七千多星，这两个项目分别代表"前端 Agent UI 框架"和"开源 NotebookLM"——加上昨天还在榜上的 superpowers、taste-skill、goose，企业内部 Agent 平台搭建的开源主力一应俱全。下半年的窗口期对所有"做 AI 应用"的团队都打开了。

主持人A：好，第一条铺完了。第二条，MiMo-v2.5-Pro-UltraSpeed 的速度突破。

主持人A：思琪，这次小米的数字到底有多夸张？

嘉宾B：非常夸张。1T 参数模型、1000 tokens 每秒的推理速度。我们做个对比——Llama 3.1 405B 在主流推理服务上的吞吐大概是每秒 100 到 200 tokens，GPT-4 / Claude 在常规配置下也在每秒 50 到 150 tokens 这个区间。小米这次直接打到 1000 tokens 每秒，是主流方案的 5 到 10 倍。HN 评论区在质疑这个数字的同时也在挖技术细节——稀疏激活、推测解码、定制硬件，三件套大概率都用了。

主持人A：技术上"超大模型 + 超低延迟"是怎么同时实现的？

嘉宾B：核心是三层叠加。第一层是稀疏激活——1T 参数不是每次推理都激活，可能是 MoE 架构、每次只激活 50-100B 子集。第二层是推测解码——用小模型做草稿、用大模型验证，把单 token 串行计算变成并行验证。第三层是定制硬件加速——可能是和国产 GPU 厂商联调的专用推理芯片，把模型权重直接烧进硬件。三件套叠加之后，确实能在某些任务上做到 1000 tokens 每秒——但代价是"通用性下降"和"硬件锁定"。

主持人A：对企业的含义？

嘉宾B：两个含义。第一，国内大模型方案在 2026 下半年的可商用性窗口正在打开。过去大家说"国内大模型方案"想的是"质量差一截、所以只能用于不重要的场景"——今天小米的数字说明，国内厂商在某些极端工程指标上已经反超。对面向中国市场的金融科技 / 保险科技公司，国内大模型的并行评估必须立即启动。第二，企业供应商管理里的"模型路由能力"维度必须加上"国内备份方案在 30 天内可切换性"作为合规和韧性指标——尤其是中美 AI 监管分叉如果在 2027 年加剧，这个能力就是续命药。

主持人A：好，第二条铺完。第三条，Microsoft 包再次被植入凭证窃取器。

主持人A：思琪，这次的攻击逻辑和昨天 Meta 事件有什么不一样？

嘉宾B：本质都是"瞄准 AI Agent 不瞄准人"，但攻击位置完全不同。昨天 Meta 的事件是 prompt injection——攻击者构造特殊输入，让客服 Agent 在"理解客户意图 → 调用内部工具"这一步越权。这是攻击的是"Agent 的语义理解层"。今天 Microsoft 的事件是包级别投毒——攻击者把恶意代码塞进 73 个看起来正常的 npm / NuGet 包，关键设计是"当 AI Agent 自动安装或解析这些包时，恶意代码立即执行并自我复制"。这次攻击的是"Agent 的依赖管理层"。

主持人A：为什么是"被 AI Agent 打开就自我复制"，而不是被人类开发者打开？

嘉宾B：因为人类开发者打开一个包通常是"装好 / 试一下 / 用某个函数"——人不会去读包里所有文件、不会自动跑 setup 脚本、有警觉性。而 AI Agent 完全不一样——Agent 在帮你"写代码 → 装依赖 → 运行测试"的过程中，会自动解析包的所有 manifest、自动跑 install hook、自动按文档跑 example。攻击者就是利用这种"Agent 的过度勤奋"——只要 Agent 试图"理解"这个包，恶意代码就会被执行。

主持人A：自我复制是怎么实现的？

嘉宾B：恶意代码一旦在 Agent 的开发环境里执行，会做三件事——一是窃取本地的凭证（GitHub token、AWS key、SSH 私钥、浏览器 cookie），二是扫描本机其他项目并把自己注入到这些项目的 package.json 里，三是发布新的恶意包到包仓库。这个循环让攻击具有蠕虫式的扩散能力——一个被感染的 Agent 会污染它接触的所有项目、然后通过开发者的发布行为把毒源扩散给更多 Agent。这是为什么 Ars Technica 这次的标题用了"self-replicating stealer"。

主持人A：这对企业 CISO 团队的直接含义？

嘉宾B：三件事。第一，传统的 SCA（软件组成分析）工具必须扩展到"Agent 调用图谱审计"——不仅要看包依赖，还要看 Agent 在装包过程中的所有动作。第二，传统 EDR 必须增加"Agent 行为基线"——一个 Agent 在正常工作里"装包 / 跑命令 / 写文件"的模式是什么，一旦偏离基线立即告警。第三，传统 IAM 必须为 Agent 单独建立身份模型——不能让 Agent 用开发者的人类身份去拉包，必须有独立的 Agent 身份与权限边界。今年下半年的 CISO 必备建设项目。

主持人A：好，第三条铺完。第四条，xAI 是不是数据中心 REIT？

嘉宾B：Martin Alderson 那篇文章的尖锐度非常高。他的论点是——xAI 正在从"前沿实验室"滑向"数据中心 REIT"。证据有三条：一是昨天 Google 月付 SpaceX 9.2 亿美元买 xAI 数据中心算力的消息；二是 xAI 最近的资本支出 80% 以上花在数据中心和 GPU 采购、模型研发占比反而下降；三是 xAI 在最近一轮融资里的估值口径开始包含"算力资产"作为关键变量，而不是"模型能力 + 用户数"。

主持人A："前沿实验室"和"算力 REIT"的估值口径差别有多大？

嘉宾B：差别巨大。前沿实验室的估值是按"未来模型代际优势 + 用户规模 + 网络效应"算的，估值倍数通常在收入的 30-100 倍，本质是"押注未来 AGI 红利"。算力 REIT 的估值是按"算力资产规模 + 长期租约稳定性 + 利率波动度"算的，估值倍数通常在收入的 5-15 倍，本质是"基础设施租赁现金流"。如果 xAI 真的滑向后者，估值会被腰斩甚至更多。

主持人A：和"AI Is Slowing Down"那篇文章是什么关系？

嘉宾B：Ed Zitron 那篇文章是 AI 怀疑论的标志性长文——主张 LLM 的边际改进正在递减、商业回报落后于投入、行业泡沫论。355 评论里有反驳也有共鸣——共鸣的人说 GPT-5 / Gemini 3 / Claude Opus 4.7 的进步幅度确实比上一代小、推理速度的提升远大于能力的提升。两条文章叠加，加上 Apple 公开"我用 Google 模型"的双层架构表态——市场对 AI 头部公司的估值口径开始出现可观测的二分。

主持人A：这对企业战略和投资团队的直接含义？

嘉宾B：AI 关键供应商风险评级——我们之前叫 AIKV——必须新增"商业模式纯度"维度。具体怎么评？三个分项指标：前沿研发占比、算力租赁占比、模型代工占比。每个供应商根据这三项的占比，分别配置"被收购概率 / 模型可替换性 / 估值口径稳定性"权重。OpenAI / Anthropic 现在仍是高纯度前沿研发派；xAI 在向算力租赁派滑；某些二线模型公司可能滑向模型代工派。这三条路线的对手方风险完全不一样，企业的尽调和合同条款都要适配。

主持人A：好，第四条铺完。第五条，Agent SKU 与开源 NotebookLM 替代。

主持人A：今天 GitHub Trending 上有几个项目都跟这个主题相关。

嘉宾B：是的。我点几个最有意思的——第一是 CopilotKit，三万四千一百多星，"The Frontend Stack for Agents & Generative UI. React, Angular, Mobile, Slack"——AG-UI Protocol 的发起者。一旦 Agent 越来越多地驱动前端界面，AG-UI 的位置类似 MCP 在工具层的位置。这是 2026 年下半年最值得跟踪的协议级标准战之一。

主持人A：第二个？

嘉宾B：第二个是 open-notebook，两万七千五百多星，"开源 NotebookLM"——今天 Google 把 Gemini 3.5 推进 NotebookLM 但只对 Ultra/企业账户开放，open-notebook 这条"开源 NotebookLM 替代"的路径变得更有现实意义。对中小企业的知识管理 / 内部研究：可以在 2-3 周内基于 open-notebook 搭出"内部 NotebookLM"私有部署版，成本是 Google Ultra 订阅的零头。

主持人A：第三个？

嘉宾B：第三个是 mvanhorn/last30days-skill，三万四千多星，"AI agent skill that researches any topic across Reddit, X, YouTube, HN, Polymarket, and the web"——多源研究合成 skill，是企业战略 / 竞品研究分析师的"日常活儿自动化"模板。配合昨天的 taste-skill（风格控制）、goose（执行四件套），Agent SKU 的"工具 / 风格 / 研究"三件套今天集齐。

主持人A：第四个是 Panniantong/Agent-Reach，覆盖中文平台。

嘉宾B：是的，两万四千多星，"Read & search Twitter, Reddit, YouTube, GitHub, Bilibili, XiaoHongShu — one CLI, zero API fees"。覆盖 Bilibili 和小红书是关键——对国内市场研究、舆情、竞品分析的 Agent 化是稀缺底层模块。海外的 last30days-skill 加上国内的 Agent-Reach，企业研究类 Agent 的"中外双语数据采集层"今天双轨打通。

主持人A：开源 Agent 栈的搭建路径现在能不能给一个总结？

嘉宾B：可以。一个企业内部 Agent 平台的开源搭建路径——主体 Agent 用 hermes-agent；记忆层用 MemPalace；执行层用 goose；研究层用 last30days-skill 加 Agent-Reach；风格层用 taste-skill；前端 UI 用 CopilotKit；知识管理用 open-notebook；编排框架用 superpowers。这套组合在 2026 上半年还是"梦"，今天在 Trending 上集齐了——任何中等规模的企业都可以在 8-12 周内完成 POC。

主持人A：好，五个话题铺完了，进入今天的企业风控与保险科技专题。

主持人A：思琪，今天我们用 5 分钟把今天的素材整理成一份"今天就该做的事"清单。

嘉宾B：好，我列了八条。

第一条，"超越聊天框"是企业内部 AI 应用的下一阶段产品形态升级窗口。今天三大平台同日交卷，背后的共识是"聊天框"不再是 AI 应用的唯一形态。对保险公司：核保改成"生成式 UI 表单 + AI 自动填空"、理赔改成"语音 + 影像多模态入口"、客服改成"Slack / 微信内嵌智能体"、风控改成"IDE 内嵌的实时反欺诈助手"。形态升级的关键不是"AI 更聪明"，是"AI 在用户当下的工作流里出现"。

第二条，AI 应用安全责任险产品立项今天新增第四类条款——"Agent 供应链投毒"。昨天 Meta Instagram 事件是 prompt injection 攻击，今天 Microsoft 73 个包是 Agent 包级别投毒。一期产品按四类风险设计独立保单——prompt injection 攻击导致的数据泄露、tool poisoning 导致的越权操作、Agent 误判导致的财务损失、Agent 供应链投毒导致的内部系统污染。被保险企业的资质要求新增"AI 包供应链审计 + Agent 调用图谱可视化 + Agent 行为基线 EDR"三层基础设施。这等于把这三种能力推成行业标准。

主持人A：第三条？

嘉宾B：第三条，AI 关键供应商风险评级（AIKV）新增"商业模式纯度"维度。今天 xAI 数据中心 REIT 叙事 + Apple 双层架构公开 + AI Is Slowing Down 长文，三个信号叠加。建议保险公司在 AIKV 里新增三个分项——前沿研发占比、算力租赁占比、模型代工占比。每个供应商根据这三项的占比配置不同的"被收购概率 / 模型可替换性 / 估值口径稳定性"权重。一线大模型供应商的对手方风险评估今年下半年必须升级。
