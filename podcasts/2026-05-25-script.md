主持人A：哈喽大家好，欢迎收听每日科技播客，我是晓宇。今天是二零二六年五月二十五号，星期一，我们用十几分钟把这两天科技圈的大事捋一遍。
嘉宾B：大家好，我是思琪。今天有一条特别燃的新闻打头——国产大模型又掀桌子了。DeepSeek 这次是降价和放大招一起来。
主持人A：你说的是 DeepSeek V4 Pro 永久降价，还有那个新发布的编码 Agent 吧？这俩今天直接霸占了 Hacker News 的前两名。
嘉宾B：对，第一名是 DeepSeek 把 V4 Pro 的降价做成永久的，五百多分；第二名是 DeepSeek Reasonix，它自己的原生编码 Agent，主打两个词——高缓存、低成本。
主持人A：高缓存、低成本，这两个词对企业意味着什么？
嘉宾B：意味着用 AI 写代码、跑自动化的单位成本又降了一截。缓存命中率高，就是同样的上下文不用反复花钱重算；成本低，就是规模化调用更划算。说白了，过去你可能因为太贵不敢全量上 AI，现在这个经济账越来越算得过来了。
主持人A：所以这不只是一条"又降价了"的新闻，而是 AI 落地的成本拐点在逼近。
嘉宾B：正是。而且它把竞争焦点很清楚地摆出来了——不只是比谁更聪明，而是比谁能让你"用得起、用得起规模"。这对国内做 AI 应用的团队是实打实的利好。
主持人A：好，那我们顺着 AI 编码这条线往下看 GitHub 热榜，今天还是它们的天下吧？
嘉宾B：完全是。我快速点几个：codecrafters 的 build-your-own-x 五十万星稳居榜首；multica 把编码 Agent 做成可管理的"数字员工"；earendil 的 pi 是个全栈 Agent 工具箱；Understand-Anything 和 codegraph 这对"代码知识图谱"组合还在涨。
主持人A：还有一个我注意到的新面孔，Anthropic 出的 knowledge-work-plugins？
嘉宾B：对，这个值得单独提一句。它不是给程序员的，是面向"知识工作者"的插件库。信号很明确：AI Agent 正在从"帮你写代码"往"帮你做各种知识工作"扩展——写报告、做调研、处理流程。对非技术部门来说，这条线很快也会落到你头上。
主持人A：听起来一片繁荣。但我记得你说今天还有一条"泼冷水"的研究？
嘉宾B：对，这条我特别想讲。arXiv 上有篇论文，标题叫《约束衰减》，研究的是 LLM Agent 在后端代码生成里的脆弱性。
主持人A："约束衰减"是什么意思？用大白话讲讲。
嘉宾B：你可以理解成：当你给 AI 的要求越来越多、约束越来越复杂——这个接口要兼容老系统、那个字段不能动、还要满足一堆业务规则——AI 一开始能记住，但随着约束堆叠，它会慢慢"顾此失彼"，可靠性下降。这就是"约束衰减"。
主持人A：这和今天前面那些"AI 写 PR"的狂欢，正好是一冷一热。
嘉宾B：对，我觉得这是今天最值得记住的一组对照。一边是工具狂飙、人人都在让 AI 写代码；一边是严肃研究告诉你——在复杂的、约束多的后端场景，AI 的输出是会"衰减"的，不能盲信。这个张力，待会儿落到保险的核心系统上，会非常关键。
主持人A：在进风控专题之前，还有一条偏硬件的，我觉得对要自建 AI 的公司很重要——AI 芯片的成本结构。
嘉宾B：对，有个数据说，内存现在已经占到 AI 芯片元件成本的近三分之二。过去大家以为 AI 的钱都花在计算核心上，其实现在大头是内存。
主持人A：这对企业的启示是什么？
嘉宾B：如果你打算自建 AI 推理的基础设施，成本测算的重心要变。内存是主导成本，这会直接影响你"到底是自己买机器自托管，还是直接调用 DeepSeek 这种便宜 API"的决策。今天这两条——DeepSeek 降价和内存成本——其实是同一道选择题的两面。
主持人A：好，那我们进入每期的核心——企业风控与保险科技专题。今天就从这道选择题开始，思琪你来给框架。
嘉宾B：好。今天的主线我概括成三句话：第一，AI 变得更便宜了，自动化降本的机会更大了；第二，AI 在复杂场景会"衰减"，所以核心系统不能盲信；第三，新的预测工具，比如群体智能，给风控开了新口子。机遇、风险、新工具，正好三块。
主持人A：先说机遇——DeepSeek 这波降价，对保险公司具体意味着什么？
嘉宾B：意味着私有化部署 AI 的门槛又降了。过去保险公司想把 AI 用在核保、理赔、客服上，要么调贵的 API，要么自建很贵的算力。现在低成本的模型加上开源的 Agent 平台，比如 earendil 的 pi、multica，你可以用比以前低得多的成本，搭一套内部的 AI 自动化。
主持人A：但你刚才也说内存成本是大头，那是不是自建不一定划算？
嘉宾B：对，这就是要算细账的地方。我的建议是：标准化、高频、对数据敏感的环节，适合自托管低成本模型；而波动大、尝鲜性质的，先用便宜 API 试。不要一上来就买一堆机器，先用 DeepSeek 这种把单位成本压到很低的服务把业务跑通。
主持人A：很务实。那风险这一面，就接着《约束衰减》那篇论文讲？
嘉宾B：对，这是今天我最想给保险同行敲的警钟。保单系统、理赔系统、交易系统，这些都是典型的"约束极多"的后端——一堆业务规则、合规要求、历史兼容。论文说的恰恰是：AI 在这种约束密集的后端代码生成里，越复杂越容易出错。
主持人A：所以结论是？
嘉宾B：结论很明确——核心系统里 AI 生成的代码，绝对不能盲信、不能"危险地跳过审查"。必须有强制的人工评审、完整的测试、全程留痕。AI 可以帮你写初稿、提效率，但在保险这种高责任领域，最后把关的一定是人。这跟我们之前几期说的"机器做模式、人守因果"是一脉相承的。
主持人A：那第三块，新工具。今天榜上那个 MiroFish 群体智能，你之前提过，今天再展开说说？
嘉宾B：好。MiroFish 是个群体智能引擎，核心是模拟大量"智能体"的群体行为来做预测，逻辑跟传统精算很不一样。传统精算靠历史数据和统计分布，成熟险种很准，但新险种、新人群、极端场景就抓瞎。
主持人A：群体智能怎么补这个短板？
嘉宾B：三个最直接的场景。第一，巨灾险的情景模拟——模拟极端气候或公共卫生事件下，大量被保险人同时触发理赔的传导路径；第二，新险种定价——没有历史数据时，用群体行为模型生成模拟数据；第三，欺诈团伙识别——欺诈团伙本身就是一个有协作的群体，群体模型比单点异常检测更能抓住这种结构性的协同。
主持人A：这三个都是传统模型的痛点，思路确实新。那今天还有一条钓鱼新闻，跟核验有关？
嘉宾B：对，骗子滥用微软的内部账号发垃圾链接。它提醒我们一个反直觉的点：核验不能只看"来源是不是可信渠道"。一个可信的通道被滥用，照样能骗到你。所以在反欺诈和身份核验上，要叠加内容信号、行为信号，而不是简单地"白名单可信即放行"。
主持人A：好，我们来做今天的总结。思琪，三个关键词？
嘉宾B：第一，DeepSeek 把 AI 的成本又压低了一截，规模化应用的经济拐点在逼近；第二，越是复杂的核心系统，越要警惕 AI 的"约束衰减"，不能盲信；第三，群体智能这类新工具，给保险精算和反欺诈开了新口子。
主持人A：给听众留一个思考：在你的业务里，哪些环节可以放心交给越来越便宜的 AI，哪些环节必须人来把最后一道关？这条线画清楚，就是你用好 AI 的关键。
嘉宾B：说得好。感谢大家的收听。
主持人A：我们明天见，拜拜。
