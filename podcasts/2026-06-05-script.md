主持人A：哈喽大家好，欢迎收听每日科技播客，我是晓宇。今天是二零二六年六月五号，星期五。昨天我们聊了 Gemma 4 12B 的预告、Project Solara、Katana BadUSB 和 Uber 配额，整个故事的关键词是"AI 从模型层下沉到系统层"。今天故事继续展开，正片来了，老规矩，十八分钟把今天最值得记住的事情捋一遍。
嘉宾B：大家好，我是思琪。今天的地图比昨天更清楚。一共五条主线——第一条还是开源大模型，Google 今天把 Gemma 4 12B 正式开源，关键词"无独立编码器的统一多模态"；第二条是 Agent 工具链，今天 GitHub Trending 前十里 Agent 相关项目占了一半以上，ECC、Hermes Agent、Supermemory、open-notebook、last30days-skill、spec-kit 集体上榜，"Agent 中台"开始成型；第三条是 Agent 经济，Uber 一千五百美元月度配额昨天才挂出来，今天 HN 头条又冒一条 kasra dot blog 用一千五百美元让大模型跑红队，"一千五百美元"成了今天的货币尺；第四条是安全，Katana BadUSB 把扬声器变成键盘，Dashlane 通报二十个加密 Vault 被盗，外设和凭据两条战线同时拉响；第五条是文档处理，PaddleOCR、MarkItDown、OpenDataLoader-PDF 同框上榜，"PDF 到结构化数据"的链路正在合流。
主持人A：开源多模态、Agent 中台、AI 货币尺、外设与凭据、文档抽取——五条都不孤立，串起来其实还是昨天那个故事的下半场——AI 产业从"模型层"完整下沉到"系统层"。咱们一条一条来。先讲分量最重的那条，Gemma 4 12B。

主持人A：思琪你先讲讲这件事的关键。
嘉宾B：好。Gemma 4 12B 的发布博客今天在 Hacker News 上排第二，热度九百九十三，评论三百六十九。Ars Technica 同期还出了一篇专门写它的——标题就一句话："Designed to run on any laptop with 16GB of RAM"。这是今天最值钱的一句话。我先说架构上的关键变化——"无独立编码器的统一多模态"。背景是这样：过去两年开源多模态主流做法是"多塔拼接"——文本走 Transformer、图像走 ViT、音频走 Whisper 之类的编码器，然后在中间层做特征对齐。这种结构的代价是部署链路长、Token 预算难算、模态间的微调耦合度高。Gemma 4 12B 把这件事彻底重写——文本、图像、音频用同一个 Transformer 主干，输入侧直接做 Patch 化，模型内部统一处理。
主持人A：12B 这个尺寸有什么特别的？
嘉宾B：12B 这一档恰好卡在"消费级硬件能跑、企业私有部署经济可行"的中间地带。"16GB 内存笔记本可运行"这句话翻译成 IT 采购语言就是——你公司里每一台两年内买的开发本，理论上都能本地跑一个通用多模态模型。再加上今天 GitHub 热榜上的 AirLLM——四 GB 单 GPU 就能跑 70B 推理——本地推理硬件价位带是两侧一起拉宽的，模型尺寸也被拉到 12B 这个甜点位。换句话说，今年下半年我们第一次看到"通用多模态 + 桌面级硬件"可以放进同一份 TCO 报价单。
主持人A：对企业意味着什么？
嘉宾B：意味着两件事。第一件，"是否要私有部署 LLM"这个问题从"理想化"变成"严肃 RFP"。保险公司的理赔影像识别、客户资料结构化、内部 RAG 搜索这三个场景，按 12B 多模态部署，硬件成本可能比 2024 年估算下降一个量级。第二件，模型选型团队的工作量变大——过去选 Claude 还是 GPT，现在要在 Claude、GPT、Gemini、Gemma、Llama、Qwen、Hermes 之间做"端到端测评 + 隐私合规分级"，每个业务线一个组合。建议立刻成立跨 IT / 信息安全 / 业务的"模型选型小组"，用一个季度把 12B 这一档的私有部署可行性研究做完。
主持人A：那 RAG 和工具调用这一侧呢？
嘉宾B：受益最大的是 OpenDataLoader-PDF 这种"AI-ready 数据"工具、MarkItDown 这种 Office 转 Markdown 工具、PaddleOCR 这种多语种文档抽取，还有 Supermemory 这种 Agent 记忆中台。当模型本身能直接吃图像、音频，预处理流水线反而变成"做减法"——少一个编码器，少一份对齐 loss，少一个集成点。这条链路的整体延迟可能下降一半。

主持人A：那咱们顺势讲第二条——Agent 工具链集体上榜。
嘉宾B：今天 GitHub Trending 的画面非常壮观。我们一个个数啊——第二名 affaan-m 的 ECC，二十万七千颗星，定位是"Agent harness 性能优化系统"；第三名 NousResearch 的 Hermes Agent，十八万颗星；第四名微软的 MarkItDown，十四万三千颗星；第五名 GitHub 官方的 spec-kit，十万八千颗星；第六名 PaddleOCR，再往下还有 D4Vinci 的 Scrapling、mvanhorn 的 last30days-skill、supermemoryai 的 Supermemory、lfnovo 的 open-notebook、AirLLM、还有 Hermes 配套的 hermes-webui——前十五名里 Agent 相关至少占十个。
主持人A：这个组合很说明问题。
嘉宾B：对。这个组合不是"又一波 LangChain 替代品"，而是 Agent 时代的"中台拼装"。我给你拆一下——Harness 这一层是 ECC，它把 Skills、Instincts、Memory、Security、Research-first 开发统一封装，对应的是"Agent 的发动机调校"；长期记忆这一层是 Supermemory，slogan 直接叫"AI 时代的 Memory API"，对应 Pinecone/Weaviate 上面那一层；研究 Skill 这一层是 last30days-skill，它把 Reddit、X、YouTube、HN、Polymarket 跨平台研究综合成一个 grounded summary；工程范式这一层是 spec-kit，GitHub 官方推的 Spec-Driven Development；形态这一层是 hermes-webui，给 Hermes Agent 配 Web 和移动端前端。
主持人A：这五层叠起来，企业内部 Agent 平台的参考架构就出来了。
嘉宾B：完全正确。2026 年企业自建 Agent 平台的路线，从"自研框架"转向"中台 SaaS 组合"。这件事对甲方意味着什么——招聘需求从"会写 LangChain 的工程师"变成"会拼装中台的架构师"。这是一个不容易倒回去的角色变化，HR 和 IT 应该联手把岗位说明书改了。
主持人A：spec-kit 这一块我想多问一句。
嘉宾B：spec-kit 是 GitHub 官方在推的 Spec-Driven Development。意思是先写规范、再让 Agent 去实现。这件事在企业合规视角下分量极重——规范文件可审计、可版本化、可回放。保险公司的核保规则、理赔规则、反欺诈规则，本来就是"规范驱动"的——这套范式天然适配。spec-kit 给的是脚手架，但内涵已经回到 1990 年代的"形式化需求工程"传统，只不过这次的"实现者"是 AI。

主持人A：第三条，Agent 经济出现可比货币尺——一千五百美元。
嘉宾B：这事儿挺有意思的。Simon Willison 的博客今天上了 HN 头条第八，热度五百七十五，评论七百零二。文章的核心是把 Uber 给员工设的一千五百美元月度 AI 配额，拆成一个行业坐标。这是首次有大型公司公开"单员工 AI 月预算"。
主持人A：一千五百美元是个什么水平？
嘉宾B：我给你算一笔账。按 Claude Sonnet 4 当前的定价，一千五百美元大约可以买到一千万到一千五百万 token 的输入，或者两百万到三百万 token 的输出。如果是 Claude Opus 4，预算大约能买到三百万到四百万 token 的输入。对一个写代码的工程师来说，这相当于每天 5 万到 7 万 token，差不多覆盖一个 mid-level 的 IC 全天用 Claude Code 写代码、Review、做 RAG 的需求。Uber 把这个数定下来，意味着大公司内部"工程师 AI 月度生产力预算"有了第一个公开锚。
主持人A：今天 HN 上还有一条 kasra dot blog 的文章，也是一千五百美元。
嘉宾B：对。kasra dot blog 的文章标题就是"I built a vulnerable app and spent $1,500 seeing if LLMs could hack it"——作者花了一千五百美元让多个 LLM Agent 跑红队任务，量化对比攻击能力与成本。今天这两条同框非常重要——一边是"一千五百美元可以让一个员工用 AI 干一个月"，另一边是"一千五百美元能让 AI 把你写的应用攻一遍"。Agent FinOps 和 AI 红队预算评估两个新赛道，今天有了第一份可参考的标价单。
主持人A：所以企业怎么用这个锚？
嘉宾B：两步走。第一，建立"Agent 用量计费"的内部看板——按团队、按业务线计量 token 调用次数、Skill 使用，月度上限可参考 Uber 的一千五百美元区间。第二，把 kasra dot blog 的实证作为采购前安全测试的预算下限——任何对外暴露的 LLM 应用，上线前应有不少于一千五百美元等价算力的红队测试预算。这两件事，本来都没有锚，今天有了。

主持人A：第四条安全。Katana BadUSB 和 Dashlane。
嘉宾B：先说 Katana BadUSB。研究员在 blog dot nns dot ee 上发了一篇文章，标题非常吸引人——"Hacking your PC using your speaker without ever touching it"。HN 热度六百三十五。技术原理是这样：BadUSB 是一种把 USB 外设伪装成键盘的攻击方式，过去需要物理接触目标机器。Katana 这个 PoC 做了一件以前被认为"不可能"的事——通过外接扬声器线圈反向供电，把音箱本身改造成一个能被远程触发的 USB HID 设备。也就是说，攻击者只要让你接一个看起来人畜无害的扬声器到电脑上，他就能在你的电脑前自动敲键盘。
主持人A：这意味着 USB 端口管控不再是全部。
嘉宾B：对。企业 SOC 过去对终端的 USB 策略大多是"块未授权存储设备 + 块网卡 + 块 HID"。但 Katana BadUSB 说明，问题不在 USB 端口本身，而在"被默认信任的外设品类"——比如音箱、键盘灯、USB 风扇。企业终端管控接下来要补的是"外设白名单 + 行为基线告警"。
主持人A：再说 Dashlane。
嘉宾B：Ars Technica 今天的报道——"Dashlane issues opaque advisory warning 20 encrypted vaults were stolen"。密码管理器 Dashlane 通报，二十个加密 Vault 被盗，但细节披露非常不透明，Dashlane 对核心问题保持沉默。"加密 Vault 被盗"这件事本身的杀伤力取决于 vault 加密强度和主密码熵——但企业风险方关心的是"披露质量"。Dashlane 这次的沉默式响应，会被监管机构作为"披露不充分"的样本拿去做案例。
主持人A：保险公司怎么响应？
嘉宾B：三件事。第一，凭据轮换演练——任何依赖第三方密码管理器的部门做一次 fire drill，确认在 24 小时内可以完成核心系统主密码替换。第二，Vendor 安全披露 SLA 重写——和密码管理器、SaaS 服务商重新签合同时加入"72 小时内披露受影响账号清单"条款。第三，把 Trivy 这种工具加入 CI/CD——今天 trivy 也在 GitHub 热榜第九，三万五千多星——做 SBOM 和 Secret 扫描兜底。这三件事的总成本远低于一次大规模凭据失窃的赔付。

主持人A：第五条本地化文档抽取——PaddleOCR、MarkItDown、OpenDataLoader-PDF。
嘉宾B：这三个项目今天一起冲到 GitHub Trending 前列——PaddleOCR 七万九千颗星，MarkItDown 十四万三千颗星，OpenDataLoader-PDF 两万三千颗星。这三个工具的合流，意味着"PDF 到结构化数据再到 LLM"的链路正在从"各家拼凑"走向"事实标准组合"。
主持人A：每一个的定位是什么？
嘉宾B：PaddleOCR 主打多语种长尾，文档里写着支持一百多种语言，国产 OCR 在保险/金融文档抽取上的看家工具；MarkItDown 是微软维护的"Office 转 Markdown"，几乎覆盖所有办公格式，是 RAG 预处理里最稳的前置组件；OpenDataLoader-PDF 是 Java 写的 PDF 解析器，主打两点——AI-ready 数据 + 自动化 PDF 可访问性合规。三个加起来，覆盖的是"任何企业 90% 的非结构化文档输入"。
主持人A：对保险业的具体影响。
嘉宾B：保险业最大的非结构化负担其实在三类文档——投保单/承保单（结构相对固定的扫描件）、理赔证明（票据、医学影像、事故记录——格式高度异构）、再保合同（长文本、合同条款）。这三类全部可以用今天上榜的三件套覆盖。对在用 ABBYY 这种闭源商业 OCR 的公司来说，2026 年是迁移到开源链路 + 私有部署 + Gemma 4 12B 这种本地多模态的窗口期。一年内不动手，三年后 IT 成本结构会被同行甩开一个身位。
主持人A：Scrapling 也在榜上。
嘉宾B：对，D4Vinci 的 Scrapling 是 Python 写的"自适应"网页爬虫，意思是页面结构变了它能自动重学，从单次请求到全量爬取统一抽象。保险公司做"被保险企业的舆情风险跟踪"——比如对承保中的中型公司做半年期舆情扫描——Scrapling 是直接可用的取数底座。Headless Chrome 还是要用，但样板代码和维护成本降一大截。

主持人A：那咱们进入今天的企业风控与保险科技专题。把今天五条主线浓缩成八个可执行建议，时间大概五分钟。
嘉宾B：好。建议一，把 Gemma 4 12B 拉进 RFP。保险公司理赔影像识别、客户资料结构化、内部 RAG 搜索三类场景，按"12B 多模态 + 16GB 笔记本/单卡推理"重新核算 TCO。今年 Q3 之前应成立跨 IT、信息安全、业务的"模型选型小组"，把 Gemma 4 12B、Llama、Qwen、Hermes 的私有部署可行性研究做完。AirLLM 这种"4GB 单卡跑 70B"方案做研究环境基线，正式生产用 Gemma 4 12B 这一档。
主持人A：建议二。
嘉宾B：凭据治理与 Vendor 风险——Dashlane 事件复盘。所有依赖 SaaS 密码管理器/凭据保管的部门必须做三件事：第一，凭据轮换演练，目标是 24 小时内完成核心系统主密码替换；第二，Vendor 安全披露 SLA 重写，要求供应商 72 小时内披露受影响账号清单；第三，CI/CD 加入 Trivy 这类工具做 SBOM + Secret 扫描兜底。
主持人A：建议三。
嘉宾B：终端外设管控——Katana BadUSB 启示。旧规则"USB 端口阻断未授权存储 + 网卡 + HID"已经不够。增加"外设白名单 + 行为基线告警"：办公区 USB 扬声器/USB 风扇/带灯键盘等"默认可信外设品类"全部纳入资产台账与签名校验。新员工入职 IT 教育中加入"USB 外设统一发放"条款。
主持人A：建议四。
嘉宾B：AI 用量治理——用 Uber 的一千五百美元做内部对标。建立"Agent 用量计费"的内部看板——按团队、按业务线计量 token、调用次数、Skill 使用，月度上限可参考 Uber 的一千五百美元区间设定。同时把 kasra dot blog 的一千五百美元红队实证作为采购前安全测试的预算下限——任何对外暴露的 LLM 应用，上线前应有不少于一千五百美元等价算力的红队测试预算。
主持人A：建议五。
嘉宾B：Agent 守则与可审计性——spec-kit 加 ECC 思路落地。保险公司若引入 Agent 化核保或 Agent 化理赔，需提前把这些做成验收硬指标：第一，Agent 操作守则文件，明确禁止/允许的工具调用；第二，Agent 行为回放，每次决策可重放、可追溯；第三，系统级权限白名单，最小权限原则；第四，Spec-Driven 模式，规范文件本身可审计、可版本化。spec-kit 给的是脚手架，内涵就是 1990 年代的"形式化需求工程"——只不过实现者换成 AI。
主持人A：建议六。
嘉宾B：监管前瞻——英国 AI Overviews 案。英国监管要求 Google AI Overviews 必须给出清晰来源链接、允许 publisher 退出。保险公司的合规 RAG 客服 Agent 立刻自检三件事：第一，是否在生成回答时给出可点击的来源文档链接；第二，是否允许"权威医疗信息源"对客服 Agent 的检索范围设置 opt-out；第三，是否保留"为什么不使用某来源"的决策日志。这三件事在英国之后，欧盟很可能跟进，国内监管也会以此为参考。
主持人A：建议七。
嘉宾B：Robotaxi 电池二次利用——B 端财产/责任险新品类。Waymo 退役动力电池进入加州和德州电网储能。意味着"动力电池二次寿命"成为可承保的财产风险与第三方责任风险。储能站起火、循环寿命远低于预期、退役电池跨州运输的责任分摊，都是新险种设计的契机。建议产险产品部门今年立项做一个"动力电池二次利用财产险"的可行性研究。
