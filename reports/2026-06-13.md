## 💼 商业化价值

**superpowers / agent-skills（Skill 包生态）**
- **市场潜力**: 高。Skill 是 Agent 时代的 SKU——可独立买卖、可版本管理、可订阅升级。一旦 Anthropic 或第三方做出 Skill Marketplace，B2B 垂直 Skill 包将爆发。
- **盈利模式**: 1) 私有 Skill 仓库订阅（SaaS）；2) 垂直行业 Skill 包销售（一次性 + 维护费）；3) Skill 审计与签名服务（合规向）。

**LMCache（KV Cache 中间件）**
- **市场潜力**: 高。当 Agent 进入"长上下文 + 高并发"阶段，KV Cache 命中率是推理成本的最大杠杆。LMCache 的"跨请求、跨节点共享缓存"是 Agent 基础设施层的关键一环。
- **盈利模式**: 开源 + 企业版（SLA、热升级、跨集群缓存协同）。

**BitBoard（Agent 协同分析）**
- **市场潜力**: 中高。BI 是企业最大的 SaaS 品类之一。把"人 + Agent"共同搭建仪表盘做成新形态，是对 Tableau、Looker 的"Agent 化重做"。
- **盈利模式**: 按席位订阅 + 数据连接器 + Enterprise SSO。

**Kimi K2.7-Code / MiMo Code（开源编程模型）**
- **市场潜力**: 中。开源编程模型本身难直接盈利，但围绕模型可以打 IDE、CI 集成、私有部署服务。
- **盈利模式**: 私有化部署 + 配套 IDE 产品 + 行业微调服务。

**FablePool（提示词众筹）**
- **市场潜力**: 中。"提示词资本化"的早期实验，更像是 Web3 时代"任务赏金"的延续。
- **盈利模式**: 平台抽成、热门 prompt 收益分润。

## 🏢 企业风控·保险场景借鉴

**机遇：把 Skills 框架引入企业风控**
- **核保 Skill 包**：把"被保人体检报告解读 + 三方数据交叉验证 + 风险等级判定"做成一个 Skill，分发给一线核保员的 Agent 工作台。参考 [phuryn/pm-skills](https://github.com/phuryn/pm-skills) 的产品化思路，**面向具体岗位封装一段端到端工作流**比给员工一个"通用 Copilot"更易落地。
- **尽调 Skill 包**：把企业征信、舆情、关联图谱、招聘动态六种来源像 [last30days-skill 思路] 一样做成"一键调研某公司近 30 天动向"的 Skill，赋能授信和反欺诈团队。
- **理赔定损 Skill 包**：参考 [Inside soccer's data renaissance](https://www.technologyreview.com/2026/06/11/1138506/inside-soccer-data-renaissance-jesse-davis/) 的"事件级 → 决策级"思路，把车险定损从图像 OCR 升级为"事件序列重建"。

**风险：Agent 失控的精算锚点正在形成**
- [Agent 扫 DN42 把运营者烧到破产](https://lantian.pub/en/article/fun/ai-agent-bankrupted-their-operator-scan-dn42lantian.lantian/) 给保险公司提供了第一个公开的精算锚点。**Agent 责任险**可参考网络安全险、专业责任险的精算方法：
  - 保费定价因子：Agent 调用频率、调用上限、人工审批比例、熔断机制是否存在、模型供应商 SLA。
  - 免赔与共保：建议对"未启用熔断"的 Agent 部署，设置高免赔；对"启用熔断 + 监控告警"的部署，给出保费折扣。
  - 排除责任：故意 prompt 注入、越权调用、对抗性输入引发的损失。

**挑战：Agent 的"隐形护栏 + 强制留存"撞上企业合规**
- [Anthropic 隐形 Fable 蒸馏护栏](https://www.theverge.com/ai-artificial-intelligence/948280/anthropic-claude-fable-invisible-distillation-guardrail) 这件事，对企业法务团队是一次提醒——和模型厂商签约时，"模型路由透明度"和"数据留存周期"要明确写进 DPA（数据处理协议）。
- 对 PII 字段（病史、身份证、地址）进入 LLM 的合规边界，应该建立"分级 Prompt 模板"——低敏字段走默认通道、高敏字段走零留存企业 SKU 或本地化模型。
- **Agent SBOM**：参考软件 SBOM 思路，企业需要建立"我的 Agent 装了哪些 Skill、每个 Skill 的版本、签名、依赖"的清单。今天 superpowers 这类框架还没有这套机制，建议提前内部建标准。
