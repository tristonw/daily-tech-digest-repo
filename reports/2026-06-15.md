## 🏢 企业风控 / 保险场景借鉴

1. **核保 / 理赔流程 Skill 化**
   把核保问询、医疗证据解析、欺诈线索复核拆成 30–80 个独立 Skill，借鉴 [agent-skills](https://github.com/addyosmani/agent-skills) 的工程结构。每个 Skill 自带输入输出 schema、可灰度回滚、可独立审计——这就把"AI 黑箱"问题转成"组件级合规问题"。
2. **「百万 Agent 互联」的反向应用：监管科技**
   DeepMind 的担忧反过来读，就是"多 Agent 行为相关性"将成为新一类系统性风险。保险公司若能率先做"AI 模型组合风险"产品（Agent Liability + Model Drift Insurance），是非常具体的承保新品。
3. **「Kage 单文件镜像」用于尽调与监管证据保全**
   反洗钱、企业风控、知识产权侵权取证场景，可以直接采用 [Kage](https://github.com/tamnd/kage) 这种纯离线镜像方案，把网页/页面证据封固成单文件二进制，链上存证后即可作为合规证据。
4. **「SQL→ER 浏览器版」启发**
   涉客户敏感字段的数据资产盘点，"不上传"本身就是卖点。保险公司内部的数据合规自助工具，可以照搬 [sqltoerdiagram.com](https://sqltoerdiagram.com/) 的纯浏览器路线。
5. **本地推理 + KV Cache 抽象用于隐私核保**
   把 GLM 5.2 / Qwen 3.6 27B 部署在本地（参考 [RTX 5080+3090 80 tok/s 方案](https://imil.net/blog/posts/2026/rtx-5080-+-rtx-3090-setup-80+-tok-s-on-qwen-3.6-27b-q8/)），同一被核保人的医疗影像、病历、问询可以走"本地 + 不出网"流水线，从根上规避 GDPR、《个人信息保护法》风险。
6. **退役设备组算力对应"绿色保险"产品**
   [Google 退役手机集群](https://research.google/blog/a-low-carbon-computing-platform-from-your-retired-phones/) 给企业碳资产、绿色 IT 投保 / 再保险 / 碳信用挂钩险种提供了新的承保因子。
7. **「AI 使用克制度」作为新风控信号**
   [No, everyone is not using AI for everything](https://gabrielweinberg.com/p/people-are-consuming-ai-like-they) 的结论提示：企业的真实 AI 渗透度远低于宣传值，承保团队若把"被保企业的 AI 使用真实强度"作为定价因子，可避免"按宣传值定价 vs 实际损失"的偏差。
