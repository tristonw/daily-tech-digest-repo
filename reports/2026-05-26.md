## 🏢 企业风控·保险场景借鉴

1. **Agent 工程化 → 把"AI 自动化"做成可治理的流水线**
   ECC、gstack、multica 这类工具的核心价值，是给 Agent 加上分工、留痕、权限与编排。保险公司在核保、理赔、客服上用 AI，不能只追求"能跑"，而要像 gstack 那样**明确每个 Agent 的角色与边界**，让自动化流程可审计、可回溯——这正是金融监管所要求的。

2. **低成本编码 Agent → 私有化降本，但要算内存账**
   DeepSeek 降价与原生编码 Agent，让保险/金融企业私有化部署 AI 的门槛进一步下探。建议：标准化、高频、数据敏感的环节适合自托管低成本模型；波动大、尝鲜性质的先用便宜 API 试。但要结合"内存主导算力成本"重新测算自托管的真实账本。

3. **群体智能（MiroFish）→ 精算与反欺诈的新工具**
   群体行为模拟适配三大痛点：新险种小样本定价（无历史数据时生成模拟数据）、巨灾情景模拟（极端事件下大量保单同时触发的传导路径）、欺诈团伙的协同模式识别（团伙本身就是协作群体，群体模型优于单点异常检测）。

4. **本地视觉（Frigate）→ 定损与安防保险**
   数据不出本地的实时检测，适合财产险现场查勘定损（设备端识别损失、规避隐私合规风险），以及"投保企业部署智能安防 → 更低风险系数与保费"的产品创新。

5. **知识图谱 + 文档管理 → 技术尽调与单证留痕**
   Understand-Anything / codegraph 把陌生系统的 IT 风险评估从数周压到数天；paperless-ngx 提供低成本的单证扫描、索引、归档底座，对中小保险中介的合规留痕极具性价比。

6. **可信渠道滥用（微软账号钓鱼）→ 供应商信任链风控**
   核验不能只看"来源是否可信渠道"。可信通道被滥用照样能骗到你，反欺诈与身份核验要叠加内容信号与行为信号，而非"白名单可信即放行"。
