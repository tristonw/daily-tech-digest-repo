## 🏢 企业风控 / 保险场景借鉴

1. **"未盈利 AI 公司被指数挡门外"是新型对手方风险信号**——S&P 500 这条规则对企业供应商风控有直接含义：当你的核心供应商是 OpenAI / Anthropic 这类"账面亏损但战略关键"的公司时，传统的"上市公司 = 低风险"路径失灵。需要建立"AI 关键供应商 (AIKV) 风险评级"——把账面亏损、烧钱速度、轮次依赖、被收购概率等纳入对手方风险矩阵，至少做到"三家以上多供应商 + 模型可替换"。

2. **Gov.uk 把 Stripe 换成 Adyen 是支付侧的"AI 时代供应商替换"演练样本**——尤其对保险公司的保费收单、理赔出账两端，"支付通道集中度风险"是显性短板。可以借鉴的工程实践：保单生命周期内的支付通道做成可插拔层，每条业务线至少签两家 PSP，定期做"通道切换演练"。

3. **rsync × Claude 的 bug 增量分析是"AI Coding 内审"的范式样本**——保险公司、银行的内部 Code Review 流程可以直接借鉴：在所有用 LLM 辅助的代码提交上打 tag，每季度做一次"AI 提交 vs 人工提交"的 bug 引入率统计。这套指标既是合规材料、也是工程改进抓手；2026 下半年大概率成为金融监管对 IT 部门的标准问询项。

4. **AI 客服 Agent 被攻击导致账号失窃——AI 应用安全责任险的入场时机**——Meta 这次事件给"AI 应用安全责任险"一个清晰锚点：保险公司可以围绕 prompt injection、tool poisoning、Agent 误操作三类风险设计独立条款。建议产品节奏：第一阶段做"事故复盘 + 蓄水池数据"，第二阶段做"垂直行业（电商客服、银行客服、医疗咨询）的专属责任险"。

5. **微软 pg_durable 给企业"工作流持久化"提供低成本路径**——理赔工作流、KYC 工作流、退保工作流这类"长时间运行、必须可恢复"的业务，过去要么自建 Temporal、要么用商业 BPM。pg_durable 让"用 Postgres 跑 durable execution"成立，对中型保险公司的 IT 选型成本大幅降低。建议把它列入"理赔中台 / 反洗钱中台"的备选基础设施。

6. **AI 数据中心 + 虚拟电厂 + SMR 这条"算力—能源"链条进入企业财产/责任险新险种孵化期**——Google 用 VPP、Antares SMR 临界测试、Google 与 SpaceX 的 9.2 亿/月算力订单，三件事拼出"AI 算力的电力依赖与基础设施集中风险"。可以围绕"超大规模数据中心电力中断责任险"、"AI 算力供应链中断险"等新险种做前期产品研究。

7. **Mouseless、CopilotKit AG-UI、VibeVoice 共同提示"Agent 操作界面 = 新的内部审计触点"**——当员工开始通过键盘短语、语音指令、Agent UI 触发企业系统操作时，传统的"按钮点击日志 + RBAC"审计完全无效。建议把"Agent 触发的业务操作"作为单独审计类目，记录三件事：触发指令原文、Agent 决策链、最终落到业务系统的动作 ID。

8. **MiroFish 群体智能 + Vibe-Trading 给"精算压力测试 + 投资组合情景仿真"提供开源底盘**——保险精算与资管部门可以把 MiroFish 这类预测引擎作为"内部对照模型"——主模型 + MiroFish 的双轨预测如果出现显著背离，作为风险预警信号。Vibe-Trading 的开源策略框架则适合做"AI Trading 风险敞口"的内部测试沙盒，不直接进实盘。

9. **PaddleOCR + 文档抽取栈对保险三大场景直接可用**——核保（保单 OCR + 结构化）、理赔（医疗票据 + 影像）、反欺诈（多文档交叉比对）。100+ 语言覆盖对跨境保险、再保险尤其关键。建议把 PaddleOCR 列入"文档智能"基础设施的开源基线。

10. **GNSS 干扰研究 + Sound Blaster 空中漏洞——给企业物理资产的"新型尾部风险"**——卫星导航被干扰、IoT 音箱被空中接管，看似两件事，本质都是"物理基础设施 / 物联终端"在 2026 年成为攻击面。对车险 UBI、智能办公园区、酒店物联资产相关险种，需要把这类"信号层 / 终端层"风险纳入定价模型与免责条款。
