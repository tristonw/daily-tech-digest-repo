## 📊 趋势分析

1. **Agent 的接入层正在从"协议"转向"技能 + 直接 API"**：MCP 在 2025 年被寄予厚望，今天 HN 一篇 [MCP is dead?](https://www.quandri.io/engineering-blog/mcp-is-dead) 引发激烈讨论；与此同时 GitHub 热榜密集出现 [anthropics/skills](https://github.com/anthropics/skills)、[ECC](https://github.com/affaan-m/ECC)、[taste-skill](https://github.com/Leonxlnx/taste-skill) 等 Skills 项目。结论是：协议之争未必能用"通用接口"一战定胜，更可能演化为"每个领域一组可分发的 Skills + 显式 API 调用"的实用主义形态。

2. **"持久化工作流"取代"模型选型"成为新的工程焦点**：[SQLite is all you need for durable workflows](https://obeli.sk/blog/sqlite-is-all-you-need-for-durable-workflows/) 高位讨论说明，团队把 Agent 投产的瓶颈从"模型够不够聪明"换到了"长任务能不能可靠重放、可审计"。本地化、嵌入式状态机正在赢回阵地。

3. **基础设施品类继续细分，并被资本逐一确认**：[OpenRouter B 轮 1.13 亿美元](https://openrouter.ai/announcements/series-b) 把"模型路由"立成一个独立的板块；加上更早的向量数据库、Agent 评测、推理加速等，AI 时代的"中间件版图"正在被一张一张地填上。

## 💼 商业化价值

- **二级开发与 Skills 市场**：[anthropics/skills](https://github.com/anthropics/skills) 与 [ECC](https://github.com/affaan-m/ECC) 这种"Skills 仓库"会催生一个新的小经济体——专门做"行业技能包"的小工作室，垂直行业（法律、医保、保险、风控）会有快速进入窗口。
- **CRM/SaaS 重做**：[twentyhq/twenty](https://github.com/twentyhq/twenty) 走的是"为 AI 设计的开源 Salesforce"路线。对 ISV、内部 IT、咨询商而言，这是一个低门槛切入"AI 原生业务系统"的底座。
- **路由与可见性服务**：OpenRouter 这一轮估值给后续做"模型路由 + 成本可观测 + 安全过滤"产品的团队定了价格锚点。企业内部一份"私有路由层"已经成为合理基础设施投入。
- **TTS 自部署**：[VoxCPM](https://github.com/OpenBMB/VoxCPM) 让"客服质检、培训语音、本地播客"这类场景脱云成为可能，对受合规约束的金融/保险特别友好。

## 🏢 企业风控·保险场景借鉴

- **可分发的"风控 Skill 包"**：Agent 落地的最大障碍是合规与一致性。把"反洗钱筛查脚本""KYC 文档抽取""被保险人病历摘要"做成可版本化的 Skill（参照 [anthropics/skills](https://github.com/anthropics/skills) 的形态），既能灰度更新，也方便审计——每条决策都能定位到"是哪一版 Skill 在哪个数据上做出的"。
- **持久化工作流是核保/理赔的天然选择**：核保从受理到出单可能持续数小时甚至数天，跨多个外部数据源。[SQLite is all you need for durable workflows](https://obeli.sk/blog/sqlite-is-all-you-need-for-durable-workflows/) 给中小险企/科技子公司一个轻量底座的范例——单实例就能跑、状态可重放，省去 Temporal/Airflow 的运维成本。
- **MCP / Skills 之争对采购的启示**：在 Agent 接入层标准未定时，建议优先"显式 API + 公司自管理 Skill 仓"的两段式结构，避免把核心业务绑死在某一种协议或某一家厂商的接入层上。
- **路由层 = 模型供应商集中度风险的缓冲**：[OpenRouter 的 B 轮](https://openrouter.ai/announcements/series-b) 也提醒企业，把"对外只签一个路由层供应商，对内并行接入多家模型"作为模型采购的标准动作。
- **AI 内容溯源 / 训练数据合规**：[家用录像换保洁](https://arstechnica.com/ai/2026/05/robot-training-startup-will-send-humans-wearing-cameras-to-clean-your-home/) 这类做法在欧盟、加州必然引发法律纠纷。保险公司应在内容审查、生成式 AI 责任险条款里提前给出训练数据来源、可撤回、二次使用等明确定义。
- **网络风险险定价**：[Botnet of 17M devices dismantled](https://arstechnica.com/security/2026/05/botnet-of-more-than-17-million-devices-dismantled/) 提示住宅代理已成为反欺诈/反爬规避的主要工具——做反欺诈服务和网络责任险定价时，需要把"住宅 IP 风险评分"作为独立维度。
- **航天与发射保险**：[New Glenn 失利](https://arstechnica.com/space/2026/05/heres-why-the-failure-of-blue-origins-new-glenn-rocket-is-so-catastrophic/) 后短期内航天巨灾再保的报价会上抬，对持有卫星资产或依赖发射服务的企业，自留风险与共保比例需要重新评估。
- **组织端的"AI 就绪度"诊断**：[MIT Technology Review 关于 Agent 化的现实检查](https://www.technologyreview.com/2026/05/26/1137584/rethinking-organizational-design-in-the-age-of-agentic-ai/) 给保险与银行内部数字化项目一个抓手——把"流程是否定义清楚、责任人是否明确、决策日志是否完整"作为部署 Agent 前的硬性入口检查。
